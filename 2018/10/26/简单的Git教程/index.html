简单的Git教程

发表于 2018-10-26

前言

相信很多人在入计科的坑前都对GitHub有所耳闻。GitHub为Git提供免费远程仓库托管服务。
Git的产生，要从Linux内核编写历程开始说起。Linux的代码编写靠的是全世界热心的志愿者，最初，是由Linus本人通过手工整合合并来自世界各地的源代码。
随着时间的发展，逐渐庞大的代码库让Linus也很难手动管理了，于是，Linus选择了一个商业的版本控制系统BitKeeper，BitKeeper的东家BitMover公司出于人道主义精神，授权Linux社区免费使用这个版本控制系统。
然后这个阖家欢喜的局面在2005年结束了，Linux社区中的一个大牛Andrew试图破解BitKeeper的协议的事情被发现了（可能之前破解的都没被发现），于是BitMover公司极其不满，要收回Linux社区的免费使用权。
于是，付费玩家Linus上线了。
他花了两周时间用效率最高的C语言写了一个分布式版本控制系统，也就是Git。一个月之内，Linux系统的源码已经由Git管理了。
Git迅速成为最流行的分布式版本控制系统，尤其是2008年，GitHub网站上线了，它为开源项目免费提供Git存储，无数开源项目开始迁移至GitHub，包括jQuery，PHP，Ruby等等。

题外话：个人非常喜欢Linus的开源精神吧。计算机行业能一直保持高速发展，和开源精神有着很大关系。GitHub某种程度上，其实把全球的程序员都连在了一起。

什么是Git

什么是Git？
Git是一个分布式版本控制软件，最初由林纳斯·托瓦兹（Linus·Torvalds）创作。最初目的是为更好地管理Linux内核开发而设计。它允许软件开发者可以共同参与一个软件开发专案，但是不必在相同的网络系统下工作。

创建版本库

什么是版本库呢？版本库又名仓库，英文名repository，你可以简单理解成一个目录，这个目录里面的所有文件都可以被Git管理起来，每个文件的修改、删除，Git都能跟踪，以便任何时刻都可以追踪历史，或者在将来某个时刻可以“还原”。
所以，创建一个版本库非常简单，首先，选择一个合适的地方，创建一个空目录：
$ mkdir MyGit
PS:mkdir=”make directory”
PS:Git对于中文的支持不是很好，命名版本库的时候避免使用中文。
$ cd MyGit
PS:cd=”change directory”
这里使用了相对路径，不懂的可以去翻一下我的hexo中关于相对路径的教程。

这样Git就把仓库建好了，而且告诉你是一个空的仓库（empty Git repository）。打开隐藏文件显示后可以发现当前目录下多了一个.git的目录，这个目录是Git来跟踪管理版本库的，不要去修改其中的内容，否则会破坏仓库。
